1. Introduction 1.1 Purpose This Product Design Document (PDD) defines the complete scope, architecture, and page-by-page specifications for the MyClean Administrator Dashboard. It is the authoritative reference for the design, engineering, and QA teams. Every endpoint listed in this document maps directly to the MyClean Backend API. 
1.2 Platform Background MyClean is an Uber-like waste collection platform connecting households and businesses in Accra with verified waste collectors (drivers). The platform operates across five revenue streams: subscriptions, on-demand pickups, driver commissions, recycling partnerships, and waste data insights. Key actors on the platform: • Users — households and businesses that book pickups • Drivers (Collectors) — verified waste collectors assigned to pickups • Admins — operations team managing the platform via this dashboard • Recycling Partners — third-party recyclers receiving sorted waste  
1.3 Problem This Dashboard Solves Without a dedicated admin interface, the operations team must query the backend database directly for every management action. The dashboard eliminates that, giving the team: • Real-time visibility into pickups, payments, and driver activity • User and driver management with action controls (suspend, reassign, approve) • Financial oversight — transactions, subscriptions, payouts — in one place • Geographic coverage monitoring across active Accra zones • Notification control for broadcast messaging to users and drivers • Full audit trail of all admin actions for accountability
